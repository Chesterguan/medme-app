## 1. 范围与非目标

### 是什么
一个 local-first 的个人医疗数据保险库:文件**进得来、存得住、找得回、看得懂、带得走**。

### 不是什么(Non-Goals)
EMR · HIS · 远程医疗 · 医院门户 · AI 诊断系统 · 预约平台 · 保险平台。

---

## 2. 核心功能

### 2.1 Import · 导入
支持格式:PDF、JPG、PNG、TIFF、DOCX、CSV、ZIP;DICOM(future)。
导入方式:拖拽、文件夹导入、扫描、拍照、Watch Folder。

### 2.2 OCR · 文字识别
自动判定:文档类型、语言、旋转、版式。
抽取(**分阶段**,见 §4):
- v0.1:整页文本(含表格结构)。
- v0.2+:诊断、化验结果、用药、操作、影像报告、过敏、生命体征等结构化实体。

### 2.3 Medical Timeline · 医疗时间线
自动生成按时间排序的医疗史。
- v0.1:以文档的临床日期(`doc_date`)为轴。
- v0.2+:以结构化临床事件(`clinical_event.event_date`)为轴,精细到"卒中 → CT → 住院 → 出院 → 用药 → 随访"。

### 2.4 Search · 搜索
搜索一切:文档、图片、时间线事件、结构化记录。
- v0.1:FTS5 全文 + 元数据(文件名、类型、日期)。
- v0.2+:按编码/概念检索(如 "Creatinine"、"Metoprolol")。

### 2.5 Export · 导出
FHIR Bundle、JSON、PDF Summary、ZIP Archive。
**用户永远可以带走自己的数据。**

## 3. 关键决策(已锁定)

| # | 决策 | 结论 | ADR |
|---|---|---|---|
| 1 | 医疗内容语言 | **中英一等公民**。Core Model 语言无关(存原文 + 归一化编码) | [ADR/0001](ADR/0001-language-cn-en.md) |

| 5 | 技术栈 | Rust + Tauri + React + TS;SQLite + FTS5 | [002_Architecture](002_Architecture.md) |

## 4. Roadmap · 路线图

> 设计文档覆盖**全部功能**;实现按版本推进。每个版本各自走 spec → plan → 实现。

| 版本 | 交付 |
|---|---|
| **v0.1** | 导入 · 本地数据库 · 时间线(按文档日期)· 搜索(FTS5)· OCR(整页文本) |
| **v0.2** | 医疗实体抽取 · 用药识别 · 化验识别 · 导出 |
| **v0.3** | FHIR 导入/导出 · 插件系统 · 医疗 AI provider |
| **v0.4** | Apple Health · Health Connect · 医院连接器 |
| **v1.0** | 稳定的 local-first 保险库 · 插件生态 · 可选云同步 |

### v0.1 验收标准(walking skeleton)
1. 拖入 PDF/图片 → CAS 存原文件、SQLite 记录、可去重。
2. 自动 OCR → 整页文本入库并建 FTS5 索引。
3. 时间线视图:文档按 `doc_date` 排列、可点开原件。
4. 搜索视图:输入关键词,命中文档/文本片段并高亮。
5. 原始文件从不被修改或删除。
6. 导出:整库 ZIP + JSON 清单,可完整带走。

---

## 5. 非功能需求
- **隐私**:默认全本地;无遥测;无强制联网。
- **可移植**:数据库格式与 CAS 布局有版本号,向后兼容(原则 6)。
- **可溯源**:任何派生数据(OCR/抽取)都记录 backend、model version、source span。
- **性能目标(v0.1)**:单份 A4 报告 OCR < 3s(onnx 后端,CPU);10 万文档量级搜索 < 300ms。
